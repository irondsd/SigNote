# Getting started 🚀

This guide walks you through running SigNote locally for development or self-hosting.

---

## Prerequisites

- **Node.js** 18 or later
- **MongoDB** — a local instance or a free [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) cluster
- **WalletConnect project ID** — get one free at [cloud.walletconnect.com](https://cloud.walletconnect.com)
- **Ethereum wallet** — MetaMask, Coinbase Wallet, or any wallet supported by RainbowKit
- An **Ethereum RPC URL** — Infura, Alchemy, or a public endpoint

---

## 1. Clone and install

```bash
git clone https://github.com/irondsd/signote.git
cd signote
npm install
```

---

## 2. Configure environment variables

Copy the example file:

```bash
cp .env.local.example .env.local
```

Then fill in `.env.local`:

```bash
NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID=""
NEXT_PUBLIC_RPC_URL=""
MONGODB_URI="mongodb+srv://<username>:<password>@<cluster-url>/"
MONGODB_DB="signote"
NEXTAUTH_URL="http://localhost:5000"
NEXTAUTH_SECRET="replace-with-a-long-random-secret"
```

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID` | WalletConnect project ID used by RainbowKit/Wagmi |
| `NEXT_PUBLIC_RPC_URL` | Ethereum RPC endpoint (Infura, Alchemy, or public) |
| `MONGODB_URI` | MongoDB connection string |
| `MONGODB_DB` | Database name |
| `NEXTAUTH_URL` | Exact public origin — **must match** what you use in the browser |
| `NEXTAUTH_SECRET` | Secret used by NextAuth to sign session tokens (make it long and random) |

> ⚠️ `NEXTAUTH_URL` is used by SIWE to validate the origin during sign-in. If it doesn't match the URL in your browser, sign-in will fail. For local dev, use `http://localhost:5000`.

---

## 3. Run the dev server

```bash
npm run dev
```

Open [http://localhost:5000](http://localhost:5000) in your browser.

---

## 4. Sign in and set up encryption

1. Click **Sign in** and connect your Ethereum wallet
2. Sign the SIWE message when prompted by your wallet
3. Once signed in, navigate to **Secrets** or **Seals** to set up encryption
4. Choose a strong passphrase (minimum 16 characters) and save your encryption profile

## Running tests

Tests use Playwright for end-to-end testing. They start a dev server and an in-memory MongoDB automatically — no manual setup needed.

```bash
npm run test                                          # Run all tests
npx playwright test tests/specs/notes.spec.ts        # Run one file
npx playwright test --ui                             # Run with Playwright UI
```

Other useful commands:

```bash
npm run lint     # ESLint + TypeScript type check
npm run format   # Prettier formatting
npm run build    # Production build
```

## Project structure

```
src/
├── app/                  # Next.js App Router pages and API routes
│   ├── api/
│   │   ├── encryption/   # Encryption profile and key material endpoints
│   │   ├── notes/        # Tier 1 CRUD
│   │   ├── secrets/      # Tier 2 CRUD
│   │   └── seals/        # Tier 3 CRUD
│   ├── secrets/          # Tier 2 pages
│   └── seals/            # Tier 3 pages
├── components/           # UI and feature components
├── config/               # Auth, wallet, and server configuration
├── contexts/             # EncryptionContext — MEK lifecycle management
├── controllers/          # MongoDB-facing application logic
├── hooks/                # TanStack Query data and mutation hooks
├── lib/
│   └── crypto.ts         # All client-side cryptographic operations
├── models/               # Mongoose schemas
└── providers/            # App-wide React providers
```

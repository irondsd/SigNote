# Self-hosting Guide 🚀

This guide walks you through running SigNote locally for development or self-hosting.

---

## Prerequisites

- **Node.js** 18 or later
- **MongoDB** — a local instance or a free [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) cluster
- **WalletConnect project ID** — get one free at [cloud.walletconnect.com](https://cloud.walletconnect.com) (required for SIWE)
- **Ethereum wallet** — MetaMask, Coinbase Wallet, or any wallet supported by RainbowKit (required for SIWE)
- An **Ethereum RPC URL** — Infura, Alchemy, or a public endpoint (required for SIWE)
- **Google OAuth credentials** (optional) — set up at [Google Cloud Console](https://console.cloud.google.com) if you want to support Google sign-in

---

## 1. Clone and install

```bash
git clone https://github.com/irondsd/signote.git
cd signote
npm install
```

---

## 2. Configure environment variables

Copy the example file:

```bash
cp .env.local.example .env.local
```

Then fill in `.env.local`:

```bash
# Required for all deployments
NEXTAUTH_URL="http://localhost:5000"
NEXTAUTH_SECRET="replace-with-a-long-random-secret"
MONGODB_URI="mongodb+srv://<username>:<password>@<cluster-url>/"
MONGODB_DB="signote"

# Required for SIWE (Sign-In with Ethereum)
NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID=""
NEXT_PUBLIC_RPC_URL=""

# Optional: for Google OAuth support
GOOGLE_CLIENT_ID=""
GOOGLE_CLIENT_SECRET=""
```

**Core variables:**

| Variable          | Description                                                              | Required |
| ----------------- | ------------------------------------------------------------------------ | -------- |
| `NEXTAUTH_URL`    | Exact public origin — **must match** what you use in the browser         | Yes      |
| `NEXTAUTH_SECRET` | Secret used by NextAuth to sign session tokens (make it long and random) | Yes      |
| `MONGODB_URI`     | MongoDB connection string                                                | Yes      |
| `MONGODB_DB`      | Database name                                                            | Yes      |

**SIWE (Sign-In with Ethereum):**

| Variable                               | Description                                           | Required |
| -------------------------------------- | ----------------------------------------------------- | -------- |
| `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID` | WalletConnect project ID from cloud.walletconnect.com | Yes      |
| `NEXT_PUBLIC_RPC_URL`                  | Ethereum RPC endpoint (Infura, Alchemy, or public)    | Yes      |

**Google OAuth (optional):**

| Variable               | Description                               | Required |
| ---------------------- | ----------------------------------------- | -------- |
| `GOOGLE_CLIENT_ID`     | OAuth 2.0 Client ID from Google Cloud     | No\*     |
| `GOOGLE_CLIENT_SECRET` | OAuth 2.0 Client Secret from Google Cloud | No\*     |

\*Only required if you want to enable Google OAuth sign-in. SIWE works without Google credentials.

### Setting up Google OAuth (optional)

If you want to support Google sign-in, follow these steps:

1. Go to [Google Cloud Console](https://console.cloud.google.com)
2. Create a new project (or select an existing one)
3. Enable the Google+ API
4. Create an OAuth 2.0 credential (type: Web application)
5. Add authorized redirect URIs:
   - For local dev: `http://localhost:5000/api/auth/callback/google`
   - For production: `https://yourdomain.com/api/auth/callback/google`
6. Copy the Client ID and Client Secret into `.env.local`

> ⚠️ `NEXTAUTH_URL` is used by both SIWE and Google OAuth to validate the origin. It must exactly match the URL you use in the browser. For local dev, use `http://localhost:5000`.

## 3. Run the dev server

```bash
npm run dev
```

Open [http://localhost:5000](http://localhost:5000) in your browser.

---

## 4. Sign in and set up encryption

1. Click **Sign in** and choose your authentication method:
   - **Sign in with Ethereum** — connect your wallet and sign a SIWE message
   - **Sign in with Google** — use your Google account (if configured)
2. Once signed in, navigate to **Secrets** or **Seals** to set up encryption
3. Choose a strong passphrase (minimum 8 characters) and save your encryption profile
4. Optional: Link additional authentication methods in your profile settings

## Running tests

Tests use Playwright for end-to-end testing. They start a dev server and an in-memory MongoDB automatically — no manual setup needed.

```bash
npm run test                                          # Run all tests
npx playwright test tests/specs/notes.spec.ts        # Run one file
npx playwright test --ui                             # Run with Playwright UI
```

Other useful commands:

```bash
npm run lint     # ESLint + TypeScript type check
npm run format   # Prettier formatting
npm run build    # Production build
```

## Project structure

```
src/
├── app/                  # Next.js App Router pages and API routes
│   ├── api/
│   │   ├── encryption/   # Encryption profile and key material endpoints
│   │   ├── notes/        # Tier 1 CRUD
│   │   ├── secrets/      # Tier 2 CRUD
│   │   └── seals/        # Tier 3 CRUD
│   ├── secrets/          # Tier 2 pages
│   └── seals/            # Tier 3 pages
├── components/           # UI and feature components
├── config/               # Auth, wallet, and server configuration
├── contexts/             # EncryptionContext — MEK lifecycle management
├── controllers/          # MongoDB-facing application logic
├── hooks/                # TanStack Query data and mutation hooks
├── lib/
│   └── crypto.ts         # All client-side cryptographic operations
├── models/               # Mongoose schemas
└── providers/            # App-wide React providers
```
